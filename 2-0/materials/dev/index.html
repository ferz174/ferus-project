ОБОЛОЧКА СИСТЕМЫ УПРАВЛЕНИЯ МОДЕЛЯМИ - TeklaStructuresModelsShell 2.0

Что находится в разработке сейчас:

  • редактирование структуры оболочки, буферных папок и т.д.
  • управление общими данными, техническими требованиями, примечаниями
  • выгрузка данных из моделей сессиями (разделенными по пользователям)
  • функционал для работы с изменениями конструкторской документации (версионность документов КД)

Что планируется в дальнейшем:

  • функции проверки и сравненения входных данных с данными экспорта по всем доп-файлам и файлам-структуры (проверка изменений)
  • дополнение конструкторских документов текстом вне программы Tekla (технические требования)
  • загрузка сторонних и любых входных данных содержащих КМД и приведение их к общей структуре (импорт)
  • функционал для работы с документацией КДД в единой структуре (автоматизированный процесс работы со стыковочными чертежами)
  • функционал для предпроектной работы для эксперта-конструктора
  • полноценный реестр компонентов
  • полноценный реестр плагинов
  • полноценный FAQ
  • Функционал раздела "Предложения и замечания"
  • полноценный раздела "Учебные материалы"
  • аутентификационный вход и доступ
  • автоматическая рассортировка моделей в зависимости от статуса и дат (Рабочая, Временная, Архив)

Что уже сделано:

  • Структурирование выходных конструкторских данных из моделей
  • Автоматическое чтение данных и файлов КД из буферных папках моделей по файлам-структуры моделей (гиперссылочность)
  • Автоопределение статуса модели. Функционал для работы с многопользовательскими серверами (доступность свободных портов)
  • Автоматическая конвертация файлов-структуры моделей в конструкторскую документацию готового вида к печати (XML-PDF)
  • Автоматическая проверка наличия файлов КД по файлам-структуры моделей в буферных папках
  • Автоматическая отправка данных из структуры буферных папок моделей в указанное назначение (экспорт)
    (в любое место: в общую папку, на сервер, в облако и т.п.)
  • Обращение к файлам не из структуры, а на стороне пользователя, что дает возможность беспрепятственной замены файлов
  • Автоматическое создание структуры на требуемых ресурсах
